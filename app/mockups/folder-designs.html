Design Rationale

Option 1: Expandable List

  • Clear hierarchy between folders and notebooks
  • Share button is separate from navigation
  • Scales well with many folders
  • Familiar pattern (like file explorers)

Option 2: Action-Focused

  • Explicit buttons remove ambiguity
  • No nested cards - cleaner visual hierarchy
  • Notebooks are abstracted (just a count)
  • Mobile-friendly card size

Option 3: Variable Height

  • Shows notebook preview without nesting cards
  • Share action clearly at folder level
  • Cards naturally vary in height based on content
  • Clickable "Manage Sharing" makes intent clear
